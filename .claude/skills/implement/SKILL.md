# Implement a Platform-Generated Project

You are implementing a project whose **requirement + solution were produced on the ai-native-dev-platform website**. The platform scaffolded a starter repo on GitHub and left the project at `IMPLEMENTATION`, waiting for you to take over. The code source-of-truth is the **GitHub repo**, not the platform's in-memory snapshot.

## Prerequisites
- The platform's MCP server is connected (tools: `get_project_status`, `get_repo_url`, `get_requirement`, `get_solution`, `run_sandbox`, `write_test_report`, `signal_deploy_verified`).
- You need the **projectId** — get it from the platform project page URL (`/projects/<id>`).

## Quick start
1. `get_project_status(projectId)` — confirm status is `IMPLEMENTATION`. If not, stop and tell the user the current stage.
2. `get_repo_url(projectId)` → `git clone` the repo; work on the local checkout.
3. `get_solution(projectId)` — the blueprint (`techStack`, `architecture`, `modules`, `dataModel`, `apiDesign`, `risks`). Cross-check `get_requirement` so no feature is missed.

## Workflows

### Implement
- Implement **module-by-module** using real file reads/edits — never blind-generate whole files.
- Follow the solution's `techStack` exactly; match the scaffold's existing style.

### Verify (the core loop — never skip)
- After each meaningful chunk, call `run_sandbox(projectId, command)` to build/test for real.
- Read the real `exitCode` / `stdout` / `stderr`. On failure, fix against the **real stack trace**, then re-run.
- Pass `fresh: true` after pushing new code, so the sandbox re-clones the latest from GitHub.
- **Never assume code works without a sandbox exit code as evidence.**

### Report tests
- When tests run, call `write_test_report(projectId, summary, testResults)` with **real** numbers.
- `failed > 0` → project stays `IMPLEMENTATION` (keep fixing). All pass → platform advances to `DEPLOYMENT`.

### Finish deploy
- The platform generates the deploy config on the website first; then verify it actually boots via `run_sandbox` (build + run).
- Pass → `signal_deploy_verified(projectId)` → project `COMPLETED`.

## Red lines
- ❌ Never set project status directly — only via `write_test_report` / `signal_deploy_verified`.
- ❌ Never paste the whole codebase back into the platform — code lives in Git; the platform stores artifacts only.
- ✅ Every "it should work" claim requires a real sandbox `exitCode` as proof.
